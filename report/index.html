Objective

To implement the RNA secondary structure prediction algorithm.

Note: Also tested our code on the different datasets.

RNA Rules

Base Pairing Rules

(No sharp turns) The ends of each pair are separated by at least 4 intervening bases i.e. if (i , j) ε S, then i < j - 4.

The elements in each pair in S consist of either {A,U} or {C,G} (in either order).

S is a matching: no base appears in more than one pair.

(No knots) If (i,j) and (k,l) are two pairs in S, then we cannot have i < k < j < l .

OPT(i,j) is the maximum number of base pairs in a secondary structure for bibi+1 … bj. OPT(i,j) = 0, if i ≥ j-4.

In the optimal secondary structure on bibi+1 …bj

  • if j is not a member of any pair, use OPT(i,j-1).
  • if j pairs with some t < j - 4, knot condition yields two independent sub-problems! OPT(i,t -1) and OPT(t+1,j-1)

Running time analysis

The problem is solved with Dynamic Programming using memoization, and involves 2D matrix OPT as mentioned before.

The recurrence relation for this problem involves requiring 2 indices to compute OPT(i,j). Hence, the number of subproblems are of the order of n^2.

The subproblems are calculated using 2 nested loops. Each subproblem involves computing max{1 + OPT(i,t-1) + OPT(t+1,j-1)} for all t satisfying the rules mentioned above. This is done using another nested loop.

the overall time complexity of the RNA Secondary Structure Computing Algorithm is O(n^3).
